# Support Chat — Дорожная карта следующих фаз

## Текущее состояние (после исправления багов)

**Работает:**
- Виджет клиента: создание чата, отправка/приём сообщений, индикатор печати, переподключение WS, восстановление после обновления страницы
- Панель оператора: авторизация, список активных чатов, архив, отправка сообщений, звуковые уведомления, индикатор печати, прочтение сообщений
- Бэкенд: Express + WS, PostgreSQL, JWT-авторизация, автозакрытие чатов по таймауту (7 мин)
- Баги исправлены: client_id INTEGER, join_chat устанавливает роль, fetchMessages без authMiddleware, автопоказ виджета

**Технический долг / проблемы:**
- Корневой `package.json` содержит `bcrypt` и `jsonwebtoken` — они нужны только в `backend/`
- Нет тестов
- Нет сидера (seed) для создания первого оператора
- Нет Docker-образа для самого приложения (только PostgreSQL)
- Секреты в `.env` с дефолтными значениями

## Фаза 2: Шаблоны ответов (Canned Responses)

Из оригинального плана (nimble-knight), этап 3.

### 2.1 БД: таблица `canned_responses`
```sql
CREATE TABLE IF NOT EXISTS canned_responses (
    id SERIAL PRIMARY KEY,
    operator_id INTEGER REFERENCES operators(id),
    shortcut VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL
);
```

### 2.2 Backend: API для CRUD шаблонов
- `GET /canned-responses` — список шаблонов текущего оператора
- `POST /canned-responses` — создать
- `PUT /canned-responses/:id` — обновить
- `DELETE /canned-responses/:id` — удалить
- Все маршруты защищены `authMiddleware`

### 2.3 Frontend: панель шаблонов в operator.html
- Кнопка «Шаблоны» в хедере или сайдбаре
- Модальное окно со списком шаблонов, поиском по shortcut/title
- Клик по шаблону — вставка в поле ввода
- Быстрый ввод: `/shortcut` + Tab или Enter в поле ввода
- CRUD интерфейс для управления шаблонами
