# План реализации улучшений для support-chat

## Текущее состояние
Базовая система чата поддержки с звуковыми уведомлениями для оператора.

## Цель
Добавить функции, характерные для топовых веб-приложений поддержки:
1. Индикатор печати в реальном времени (typing indicators)
2. Индикатор прочтения сообщений (read receipts)
3. Шаблоны ответов для операторов (canned responses)
4. Горячие клавиши для панели оператора

## Последовательность реализации

### Этап 1: Индикатор печати (Высокий приоритет)
#### Backend (server.ts):
- Добавить обработку событий "typingStart" и "typingStop" через WebSocket
- Расширить функцию safeSend для broadcast typing событий участникам чата
- Не сохранять typing события в БД (только реальное время)

#### Frontend (operator.js):
- Добавить элемент UI для отображения индикатора печати в chatHeader
- При вводе текста в input отправлять "typingStart" с debounce (300мс)
- При паузе ввода более 1с отправлять "typingStop"
- При получении typing события показывать/скрывать индикатор с анимацией

#### Frontend (widget.js):
- Аналогично operator.js для виджета клиента
- Показывать индикатор печати оператора в заголовке виджета

### Этап 2: Индикатор прочтения сообщений (Высокий приоритет)
#### Backend:
- Добавить поле `read_at` в таблицу messages (или отдельную таблицу message_reads)
- При получении сообщения помечать как прочитанное для получателя при открытии чата
- Вести счетчик непрочитанных сообщений в чате
- Broadcast события "messageRead" при изменении статуса прочтения

#### Frontend:
- Отправлять событие "messageRead" при прокрутке чата к новому сообщению
- Отображать двойную галочку или статус "Прочитано" рядом с сообщениями
- Показывать счетчик непрочитанных сообщений в списке чатов

### Этап 3: Шаблоны ответов (Средний приоритет)
#### Backend:
- Добавить таблицу canned_responses с полями: id, operator_id, category, shortcut, content
- API endpoints: GET /canned-responses, POST /canned-responses, PUT /:id, DELETE /:id

#### Frontend (operator.js):
- Добавить панель шаблонов ответов в боковую панель или модальное окно
- Поддержка поиска и фильтрации по категориям
- Вставка шаблона в input по клику или через горячие клавиши (например, /shorcut)
- Возможность создания новых шаблонов прямо из панели оператора

### Этап 4: Горячие клавиши (Средний приоритет)
#### Frontend (operator.js):
- Ctrl + Enter: Отправить сообщение
- Ctrl + K: Фокус на поиск чатов
- Ctrl + Shift + M: Переключить режим mute звуков
- Ctrl + L: Отметить все сообщения как прочитанные
- Escape: Закрыть модальные окна/панели
- Up/Down: Навигация по истории отправленных сообщений в input
- Ctrl + /: Показать справку по горячим клавишам

## Технические детали
- Все новые функции должны быть обратно совместимы
- Добавить типы TypeScript для новых событий WebSocket
- Оптимизировать количество broadcast сообщений (использовать комнаты)
- Добавить обработку ошибок и edge cases (например, быстрый ввод/удаление)
- Учесть политику автовоспроизведения браузеров для звуков при новых функциях
